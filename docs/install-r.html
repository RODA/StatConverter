R installation and necessary packages

R on the system PATH

On MacOS and Linux, R is automatically added to the system PATH.On Windows, this must be done manually:

  • search for "Edit the system environment variabled" in Control Panel

  • click the "Advanced" tab

  • click on the "Environment Variables…" button

  • double-click on the "Path" variable to open it

  • click on "New", then "Browse" and indicate the folder where R is installed, typically in:

    C:/Program Files/R/R-4.2.0/bin

    (the actual version number depends on the moment when R is installed)

Necessary R packages

StatConverter uses a couple of R packages that need to be installed. The following command installs them: 

install.packages(c("DDIwR", "jsonlite"), dependencies = TRUE)

It is important to have these packages with all their dependencies, otherwise functionality might be lost.

The actual package that does the heavy lifting is DDIwR (DDI with R), which uses the package haven which in turn uses Evan Miller's ReadStat C library.

 

Updating the R packages

If you have your own R, the necessary packages will change versions and future functionality of StatConverter might depend on future version of the R packages, especially DDIwR that relies on a number of other packages, among which admisc and declared.

 

The official versions of the packages are located on CRAN, and they can be re-installed using the above command.

 

The very latest development versions of the packages are located on GitHub. To install those, you need to instruct R on using an additional package management server: 

options(repos = c("dusadrian.r-universe.dev", getOption("repos")))
install.packages("DDIwR", dependencies = TRUE)

 

For detailed instructions, visit Adrian Dușa's R-universe repository.
