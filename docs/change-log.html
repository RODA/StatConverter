Change log

A summary of the most important changes in StatConverter. Platform-specific packaging tags are omitted here in favor of the main application milestones.

 

Version 1.3.2 (current development line)

Current work focuses on improving the conversion workflow and polishing release artifacts. Notable recent changes include:

  • added a harmonize option synchronized with ongoing DDIwR changes
  • improved the speed of metadata collection
  • refined binary naming and download links
  • updated versioning and release automation files

 

Version 1.3.x maintenance line

  • embedded a portable R runtime directly with the application
  • corrected portable R links and architecture labels
  • updated package metadata and published download links

 

Version 1.3.0

  • introduced a cross-platform build dispatcher behind npm run dist
  • added a dedicated Linux build script
  • split artifact renaming into separate scripts for Linux, macOS, and Windows
  • improved Windows packaging, including executable directory zipping and spawn fixes
  • updated npm dependencies, the webR library, and supporting documentation

 

Version 1.2.1

  • reduced the bundled R library size substantially
  • continued Linux compatibility work and general cross-platform improvements
  • added pre-distribution environment checks for production builds
  • started preparing the auto-update workflow

 

Version 1.2.0

  • migrated the application to use webR
  • refreshed the application instructions and download assets
  • temporarily disabled update checks during the transition

 

Version 1.1.9

  • improved the Windows release flow and packaging variants
  • updated download links and application descriptions
  • continued packaging cleanup for the 1.1.x line
